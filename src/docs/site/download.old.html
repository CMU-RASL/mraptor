Version 1.2.3-1

  • Changelog:
    • This release primarily updates the graphical client (claw).
    • This list is a summary; for a detailed changelog, see the first section of todo.txt.
    • Process output filtering:
      • Now off by default
      • When the filtering preferences is toggled, immediately hide/show the filter section of all open process widgets.
      • Widened the default filter entry.
      • For reference, Ctrl+U clears the contents of an entry box.
      • The filter entry now remains visible when a process is dead, to enable filtering of process output after a crash.
      • Added history to the filter, akin to the process input history. Use the up/down arrow keys to utilize it.
      • An individual filter can now be enabled/disabled without editing the text, via a checkbox to the right of the filter entry, or via a context menu available by right-clicking the filter.
      • The filter entry's background now changes color based on the filter state (white = inactive; red = active, but editing in progress; green = active).
      • Fixed focus/expansion behavior; it now works properly.
      • Added context-matching: right-click a filter entry, and select "Set Context..." to select how many lines prior to and after each match to print.
    • Added help menu, which points to a localized copy of the www.microraptor.org site.
    • Made a variety of user interface cleanups, such as using checkboxes in the preferences dialog as appropriate, colorizing the local user's entries in the chat dialog, and creating a Preferences menu to provide quick access to the most common Claw preferences.
    • Added buffer reset/clear to the process context menu.
    • Claw now parses the standard --geometry argument.
    • Removed "get status" from Claw, as it is no longer needed. mraptord still supports the command.
    • Bugfixes:
      • Multiple timestamps in very long lines no longer occur.
      • Resolved a couple of minor sysv init script errors.
      • Resolved process context menu bug when viewed in a group.
      • Resolved bug that forced the user to double-click on the signals context menu to select an item.

Version 1.2.0-1

  • Changelog:
    • Added debugger support: set the 'debugger' field to 'gdb' to run the process under gdb. The arguments will be loaded automatically; set any breakpoints, then enter 'run' to start the process.
    • Added client-side output filtering to claw, via POSIX regexps. This may be disabled by setting the client-side 'stdout_filtering' preference to 0.
    • Added ability to send arbitrary signals to processes.
    • Added limited chat support to claw and mrterm.
    • Claw no longer locally echoes process status messages; they are now provided by the daemon. As a result, stdin from all clients is echoed to all clients.
    • Added dialog during connection to central to make incorrect centralhost configs clear. Also added dialog reporting central crashes, before claw exits.
    • Added stdin commands: these are lines that will be sent to the process as soon as it's launched.
    • Added (optional) timestamping to client output. This is controlled via the client-side 'display_timestamps' preference.
    • Added commandline and preferences option to specify the default width of the configuration dialogs in claw. (--config-width / default_config_width)
    • Added backup kill cmd and kill timeout to process configs (claw).
    • Made multiline entries in process config dialog expand with the window (claw).
    • Made rate limits command line arguments for mraptord, and added relevant options to mraptord.conf.
    • Added taskbar icon for claw.
    • Fixed bug in which hiding, then re-viewing a process didn't fill the buffer with recent output (claw).
    • Fixed bug in which syntax errors when entering config changes cleared the relevant field (claw).

  • Changelog:
